Data spaces

Trusted exchange of metrology information

In this project, a data space means agreed rules for describing, sharing and using information between organisations while preserving control over sources, access scope and metadata quality.

This understanding of a data space refers to the European approach to data reuse, Common European Data Spaces and principles introduced by the Data Governance Act.

Elements of a data space

The project organises both technical resources and organisational rules needed for responsible data use.

Resources

Data catalogue

Description of datasets, documents, services and registers related to metrology and quality infrastructure.

Standards

Interoperability

Agreed concepts, formats, identifiers and minimum requirements for information exchange.

Metadata

Context and quality

Information on source, currency, responsibility, applicability and limitations of data.

Trust

Access and accountability

Rules for information sharing, user roles and transparent conditions for resource use.

Simplified information flow

Data sources

Institutions, laboratories, registers, documents and domain systems.

Metadata

Description of meaning, quality, responsibility and conditions of use.

Sharing

Controlled access, cataloguing and information exchange mechanisms.

Use

Analyses, public services, administrative decisions and laboratory processes.

Example use cases

  • Finding the appropriate metrology service and responsible organisation.
  • Linking a calibration certificate with traceability information.
  • Comparing quality requirements across regulatory domains.
  • Preparing data for reporting and strategic analysis.
  • Using administrative and statistical data to assess the impact of metrology.

Importance for the project

A data space is not a single database. It is an agreed mode of cooperation that allows institutions to retain control over information while increasing its public value.

The project assumes cooperation between separate stakeholder systems, based on shared standards, metadata, secure data reuse and techniques such as anonymisation and pseudonymisation.
